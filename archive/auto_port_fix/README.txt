================================================================================
АВТОМАТИЧЕСКОЕ ОПРЕДЕЛЕНИЕ ПОРТА - ИНСТРУКЦИЯ
================================================================================

ПРОБЛЕМА:
  Ошибка "[Errno 98] Address already in use" - порт занят другим процессом.

РЕШЕНИЕ:
  Автоматический поиск свободного порта при запуске сервера.

================================================================================
КУДА ПОЛОЖИТЬ ФАЙЛЫ:
================================================================================

1. main.py
   → Положить в: /app/backend/main.py
   → Или:         /app/main.py
   → (заменить существующий файл)

2. main_with_graceful_shutdown.py
   → Вариант с graceful shutdown (рекомендуется для production)
   → Переименовать в main.py и положить в: /app/backend/main.py

3. start.sh
   → Положить в: /app/start.sh
   → Сделать исполняемым: chmod +x /app/start.sh

4. port_finder.py (опционально)
   → Утилита для тестирования
   → Можно положить в любую директорию

================================================================================
БЫСТРОЕ РЕШЕНИЕ (одна команда):
================================================================================

# Заменить main.py в вашем проекте:
cp main.py /app/backend/main.py

# Или если структура другая:
cp main.py /app/main.py

================================================================================
КАК ЭТО РАБОТАЕТ:
================================================================================

1. Проверяется переменная окружения PORT (для облачных платформ)
2. Проверяется, свободен ли порт
3. Если занят - автоматически ищется следующий свободный (8001, 8002...)
4. Исключаются порты других сервисов (3000, 12600, и т.д.)

================================================================================
ПРОВЕРКА:
================================================================================

# Проверить какие порты заняты:
netstat -tlnp

# Проверить конкретный порт:
lsof -i :8000

# Убить процесс на порту (если нужно):
fuser -k 8000/tcp

================================================================================
ЛОГИ ПРИ ЗАПУСКЕ:
================================================================================

Если порт свободен:
  ✅ Порт 8000 свободен
  🎯 Запуск сервера на 0.0.0.0:8000

Если порт занят:
  ⚠️ Порт 8000 занят, поиск свободного порта...
  ✅ Найден свободный порт: 8001
  🎯 Запуск сервера на 0.0.0.0:8001

================================================================================
